# PayNest – Smart Expense Sharing

PayNest is a Splitwise-style expense sharing web app built with FastAPI + Supabase + Vanilla JS.

## Features
- Google OAuth-style login flow (frontend scaffold + backend user upsert/login endpoint)
- Group creation, membership, expense logging, settlements, balances
- Split methods: equal, percentage, exact, share-based
- Realtime updates using Supabase Realtime subscriptions
- Web notifications for expense/settlement/member events
- Analytics charts using Chart.js
- Responsive layout for desktop/tablet/mobile
- Coming soon receipt scanner page

## Backend setup
1. Create a Python virtualenv.
2. Install dependencies:
   ```bash
   pip install -r backend/requirements.txt
   ```
3. Create `.env` inside `backend/`:
   ```env
   SUPABASE_URL=your_supabase_project_url
   SUPABASE_ANON_KEY=your_supabase_anon_key
   JWT_SECRET=super-secret-key
   ```
4. Start API:
   ```bash
   uvicorn main:app --reload --host 0.0.0.0 --port 8000
   ```

## Frontend setup
Serve `frontend/` statically (any static server):
```bash
python -m http.server 5500
```
Open `http://localhost:5500/index.html`.

Set these keys in `localStorage` from browser console before login/auth + realtime:
```js
localStorage.setItem('supabaseUrl', 'https://your-project.supabase.co');
localStorage.setItem('supabaseAnonKey', 'your-anon-key');
localStorage.setItem('apiBase', 'http://localhost:8000');
```

Authentication behavior:
- Email login/signup uses Supabase Auth (`signInWithPassword` / `signUp`).
- Google login uses Supabase OAuth (`provider: 'google'`).
- After Supabase auth succeeds, frontend calls backend `POST /login` to upsert user + receive backend JWT for protected APIs.

## Supabase SQL schema
```sql
create table if not exists users (
  id uuid primary key default gen_random_uuid(),
  name text not null,
  email text unique not null,
  photo text,
  created_at timestamptz default now()
);

create table if not exists groups (
  id uuid primary key default gen_random_uuid(),
  name text not null,
  created_by uuid references users(id),
  currency text not null default 'INR',
  created_at timestamptz default now()
);

create table if not exists group_members (
  group_id uuid references groups(id) on delete cascade,
  user_id uuid references users(id) on delete cascade,
  role text default 'member',
  joined_at timestamptz default now(),
  primary key (group_id, user_id)
);

create table if not exists expenses (
  id uuid primary key default gen_random_uuid(),
  group_id uuid references groups(id) on delete cascade,
  title text not null,
  amount numeric not null,
  paid_by uuid references users(id),
  date date not null,
  description text,
  receipt_url text,
  created_at timestamptz default now()
);

create table if not exists expense_splits (
  id uuid primary key default gen_random_uuid(),
  expense_id uuid references expenses(id) on delete cascade,
  group_id uuid references groups(id) on delete cascade,
  user_id uuid references users(id),
  owed_amount numeric not null,
  created_at timestamptz default now()
);

create table if not exists settlements (
  id uuid primary key default gen_random_uuid(),
  group_id uuid references groups(id) on delete cascade,
  payer uuid references users(id),
  receiver uuid references users(id),
  amount numeric not null,
  date date not null,
  created_at timestamptz default now()
);
```

## Vercel deployment (fix for 404 NOT_FOUND)
This repository keeps static pages under `paynest/frontend/`, so Vercel needs routing rules at the repo root.

- Keep `vercel.json` at repository root (already added in this project).
- After pulling latest changes, redeploy from Vercel.

The routes map:
- `/` -> `paynest/frontend/index.html`
- `/<page>` -> `paynest/frontend/<page>.html`
- static assets (e.g. `/css/*`, `/js/*`) -> `paynest/frontend/...`

## Notes
- Frontend uses Supabase Auth for email/password and Google OAuth login. Ensure redirect URL in Supabase Auth includes your deployed `index.html` URL.
- JWT is generated by backend and stored in localStorage.
- Enable Supabase Realtime for `expenses`, `settlements`, and `group_members` tables.
